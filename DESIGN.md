# TideTrading Front-end Visual Identity System (DESIGN.md)

This document defines the strict visual identity system and visual contract for the **TideTrading (潮汐投研)** user interfaces. Every React component, styling chunk, and layout generated by AI must無条件遵守 these rules to ensure premium, data-dense, hacker-terminal style visual excellence.

---

## 🪐 1. Theme & Atmosphere (视觉调性)
- **Style**: **"Hacker Terminal meets Bloomberg Terminal"**. Extremely data-dense, compact, flat, dark-mode focused dashboard.
- **Elevation**: Completely flat. **Zero shadows (`box-shadow: none`)**.
- **Boundaries**: Delimited by extremely thin, low-opacity borders rather than shadows or thick margins.
- **Page Layout**: Grid-based multi-card layouts, tight paddings (e.g., `p-2`, `p-3`, `p-4`), maximizing vertical and horizontal space utilization.

---

## 🎨 2. Color Palette & Semantics (色彩与对比度)

All UI elements must utilize **OKLCH** colors or structured hex tones. Middle gray/pure black is forbidden.

---

## 🔡 3. Typography & Alignments (字体排版)
- **Headers & Labels**: Choose **DM Sans** or **Geist** font families. Max 2 font families per page.
- **Financial Numbers**: **Tabular Figures Monospace** is mandatory for all numerical statistics (prices, volume, changes, timestamp columns). Use:
  ```css
  font-variant-numeric: tabular-nums;
  font-family: 'JetBrains Mono', 'Fira Code', monospace;
  ```
  *Why*: Prevents horizontal layout jitter or alignment pulling when numbers refresh rapidly.

---

## 📦 4. Spacing & Grid System (间距与排网)
- **Base Grid**: Built strictly on **4px / 8px** modular padding/margin systems (4/8/12/16/24/32px).
- **Margins**: Zero out child containers margins, rely entirely on parent layout gaps:
  ```tailwind
  className="flex flex-col gap-2 p-3"
  ```
- **Container Nesting (No直角直通)**: Parent `ResponsiveGridLayout` manages borders and glass effects. Child card components inside must be transparent (`bg-transparent border-0 rounded-none w-full h-full`) to prevent ugly double borders or dark straight corners.

---

## ⚡ 5. Transitions & Motions (动效约束)
- **Duration**: Micro-interactions (hover, click toggles) must complete within **100-200ms**. Multi-step transitions within **300-500ms**.
- **Easing**: Utilize standard cubic-bezier:
  ```css
  transition: all 200ms cubic-bezier(0.16, 1, 0.3, 1);
  ```
- **Constraint**: **Zero bounce or elastic easing** (looks cheap and lags terminal-themed UI).

---

## 🔍 6. Visual Validation SOP (强制走查)
Before committing any UI modifications, the Agent must:
1. Compile the app via `npm run build` to verify there are zero TypeScript/Tailwind static failures.
2. Trigger the `browser_subagent` to render the interface on the test server (`:9888` or `:19997`).
3. Screenshot the viewport, review the layout visually (verify borders, zero overlaps, correct tabular numbers alignment), and include the screenshot in the `walkthrough.md` report.
